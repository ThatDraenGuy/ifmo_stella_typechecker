## Общие примечания по реализации
1. Сообщения о найденных ошибках представляются в следующем общем формате:
    ```text
    КОД_ОШИБКИ:
    Строка y1, Столбец x1 - Строка y2, Столбец x2
    <исходный текст по этим "координатам" (выражение с ошибкой)>
    <Описание ошибки>
    ```
    Например, на данную программу:
    ```stella
    language core;
    extend with #sum-types;
    
    
    fn main(n : Nat + Bool) -> Nat {
      return match n {
        inl(num) => num
      }
    }
    ```
    Будет выведено:
    ```text
    ERROR_NONEXHAUSTIVE_MATCH_PATTERNS:
    Строка 6, Столбец 9 - Строка 8, Столбец 2
    match n {
        inl(num) => num
      }
    Match-выражение не покрывает все возможные паттерны.
    Список непокрытых паттернов: 
    inr(__something__)
    ```

2. Тесты вынесены в отдельный [сабмодуль](https://github.com/ThatDraenGuy/stella_test_suite/tree/main), так и не знаю, подключил ли его кто-нибудь из однокурсников)

## Запуск/сборка тайпчекера
Простейший способ - `make run`. Собирает docker-образ при его отсутствии и запускает его в интерактивном режиме.

"Натуральным" для баша образом можно подставить файл в качестве stdin для тайпчекера:
`make run < samples/ill-typed.stella`

Также есть команда `make run-rebuilt`. Работает аналогично `make run`, но обязательно пересобирает докер-образ

Разумеется, можно собрать и запустить тайпчекер напрямую через докер без мейка - `docker compose run stella-typechecker`

Можно собрать проект напрямую без докера простым `mvn package -DskipTests`. Пропустить тесты необходимо, если при клонировании репозитория не был склонирован гит-сабмодуль с тестовыми файлами.
